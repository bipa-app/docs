## Introduction

The Balances API provides real-time access to customer account balances and transaction history. Each customer can hold balances in multiple currencies simultaneously.

## Balance types

Each currency has two balance components:

| Type | Description |
|------|-------------|
| `available` | Funds available for withdrawal or trading |
| `pending` | Funds awaiting confirmation (deposits, pending trades) |

**Total balance** = `available` + `pending`

## The balance object

```json
{
  "customer_id": "cus_a1b2c3d4e5f6",
  "balances": {
    "BRL": {
      "available": "150000",
      "pending": "0",
      "currency": "BRL",
      "decimals": 2
    },
    "USDT": {
      "available": "245.67",
      "pending": "100.00",
      "currency": "USDT",
      "decimals": 2
    },
    "BTC": {
      "available": "0.05234567",
      "pending": "0.01000000",
      "currency": "BTC",
      "decimals": 8
    }
  },
  "updated_at": "2024-01-15T10:30:00Z"
}
```

### Balance attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `available` | string | Available balance (string for precision) |
| `pending` | string | Pending balance |
| `currency` | string | Currency code |
| `decimals` | integer | Decimal places for display |

## Currency precision

| Currency | Decimals | Example |
|----------|----------|---------|
| BRL | 2 | `"150000"` = R$ 1,500.00 (stored as cents) |
| USDT | 2 | `"245.67"` = 245.67 USDT |
| BTC | 8 | `"0.05234567"` = 0.05234567 BTC |

<Note>
  BRL balances are stored in **centavos** (cents). Divide by 100 for display.
</Note>

### Transaction attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | string | Unique transaction identifier |
| `customer_id` | string | Customer who owns this transaction |
| `type` | string | Transaction type |
| `currency` | string | Currency of the transaction |
| `amount` | string | Transaction amount |
| `balance_after` | string | Balance after this transaction |
| `direction` | string | `credit` (increase) or `debit` (decrease) |
| `description` | string | Human-readable description |
| `reference_id` | string | Related object ID |
| `reference_type` | string | Related object type |
| `created_at` | string | ISO 8601 timestamp |

## Transaction types

| Type | Direction | Description |
|------|-----------|-------------|
| `pix_deposit` | credit | Incoming Pix payment |
| `pix_withdrawal` | debit | Outgoing Pix payment |
| `trade` | credit/debit | Trade execution (both sides) |
| `onchain_deposit` | credit | On-chain crypto deposit |
| `onchain_withdrawal` | debit | On-chain crypto withdrawal |
| `lightning_deposit` | credit | Lightning invoice paid |
| `lightning_withdrawal` | debit | Lightning payment sent |
| `fee` | debit | Service fee charged |

## Balance updates

Balances update in real-time for:
- Completed Pix payments
- Executed trades
- Confirmed on-chain deposits
- Paid Lightning invoices

Use webhooks to react to balance changes without polling.
