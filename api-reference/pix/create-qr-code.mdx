## Request

Generate a Pix QR code that can be scanned to send payments to a customer.

### Body parameters

<ParamField body="customer_id" type="string" required>
  Customer who will receive the payment
</ParamField>

<ParamField body="amount_cents" type="integer">
  Fixed amount in cents. If omitted, payer can enter any amount.
</ParamField>

<ParamField body="description" type="string">
  Description shown to payer (max 140 characters)
</ParamField>

<ParamField body="expiration_seconds" type="integer" default="3600">
  Seconds until the QR code expires (max 86400 = 24 hours)
</ParamField>

## Response

<ResponseField name="id" type="string">
  Unique QR code identifier
</ResponseField>

<ResponseField name="customer_id" type="string">
  Customer who will receive payments
</ResponseField>

<ResponseField name="type" type="string">
  `static` (no amount) or `dynamic` (with amount)
</ResponseField>

<ResponseField name="amount_cents" type="integer">
  Fixed amount, if specified
</ResponseField>

<ResponseField name="payload" type="string">
  The Pix payload string (EMV format)
</ResponseField>

<ResponseField name="qr_code_base64" type="string">
  Base64-encoded PNG image of the QR code
</ResponseField>

<ResponseField name="expires_at" type="string">
  ISO 8601 expiration timestamp
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 creation timestamp
</ResponseField>

```python Python
import requests
import base64

response = requests.post(
    "https://api.bipa.app/v1/pix/qr-codes",
    headers={
        "Authorization": "Bearer sk_live_...",
        "Content-Type": "application/json"
    },
    json={
        "customer_id": "cus_a1b2c3d4e5f6",
        "amount_cents": 50000,  # R$ 500.00
        "description": "Order #12345",
        "expiration_seconds": 3600
    }
)

qr = response.json()

# Save QR code image
if qr.get("qr_code_base64"):
    img_data = qr["qr_code_base64"].split(",")[1]
    with open("pix_qr.png", "wb") as f:
        f.write(base64.b64decode(img_data))
```

```javascript Node.js

const img = document.createElement("img");

```

## QR code types

### Dynamic QR (with amount)
- Fixed payment amount
- Single use
- Expires after payment or timeout
- Best for: invoices, e-commerce checkout

### Static QR (without amount)
- Payer enters amount
- Reusable
- Expires only by timeout
- Best for: donations, tips, general receiving

## Displaying the QR code

You can display the QR code in two ways:

### 1. Using the base64 image

```html
<img src="{qr_code_base64}" alt="Pix QR Code" />
```

### 2. Using the payload with a QR library

```javascript
import QRCode from "qrcode";

// Generate your own QR from the payload
const qrDataUrl = await QRCode.toDataURL(payload);
```

## Handling payments

When someone pays the QR code, you'll receive a `pix.payment.received` webhook:

```json
{
  "id": "evt_abc123",
  "type": "pix.payment.received",
  "data": {
    "object": {
      "id": "pix_pay_xyz789",
      "customer_id": "cus_a1b2c3d4e5f6",
      "qr_code_id": "pix_qr_def456",
      "amount_cents": 50000,
      "status": "completed",
      "payer": {
        "name": "Maria Santos",
        "document": "***456***"
      }
    }
  }
}
```

<Tip>
  Use the `qr_code_id` to match incoming payments to specific orders or invoices.
</Tip>
