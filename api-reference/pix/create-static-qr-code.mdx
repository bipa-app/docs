title: "Create Static Pix QR Code"

api: "POST https://api.bipa.tech/v1/pix/qr-code/static"
---

## Request

Generate a Pix QR code that can be scanned to send payments to a customer.
We will only process payments from a sender that have the same document as the customer.
If the sender has a different document, the payment will be rejected.
If you want to send a payment from a sender with a different document, talk to your account manager.

### Body parameters

<ParamField body="customer_id" type="string" required>
  Customer who will receive the payment
</ParamField>

<ParamField body="amount" type="string">
  Fixed amount in centavos (e.g., `"50000"` for R$ 500.00). If omitted or `"0"`, payer can enter any amount.
</ParamField>

<ParamField body="reconciliation_id" type="string">
  Reconciliation ID for the payment. This is an input your user will provide to identify the payment.
</ParamField>

## Response

<ResponseField name="id" type="string" required>
  Unique QR code identifier
</ResponseField>

<ResponseField name="customer_id" type="string" required>
  Customer who will receive payments
</ResponseField>

<ResponseField name="amount" type="string" required>
  Fixed amount in centavos, if specified
</ResponseField>

<ResponseField name="decimals" type="integer" required>
  Decimal places (always 2 for BRL)
</ResponseField>

<ResponseField name="br_code" type="string" required>
  Pix payload (BR Code) for the QR code. Contains string `br.gov.bcb.pix`.
</ResponseField>

<ResponseField name="reconciliation_id" type="string">
  Reconciliation ID for the payment. This is an input your user will provide to identify the payment.
</ResponseField>

<ResponseField name="qr_code_base64" type="string" required>
  Base64-encoded PNG image of the QR code with the partner logo in the center.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 creation timestamp
</ResponseField>

```python Python
import requests
import base64
from decimal import Decimal

def to_centavos(amount: Decimal) -> str:
    return str(int(amount * 100))

response = requests.post(
    "https://api.bipa.tech/v1/pix/qr-code/static",
    headers={
        "Authorization": "Bearer bipa_live_sk_...",
        "Content-Type": "application/json"
    },
    json={
        "customer_id": "cus_a1b2c3d4e5f6",
        "amount": to_centavos(Decimal("500.00")),  # R$ 500.00
        "reconciliation_id": "1234567890"
    }
)

qr = response.json()
amount_brl = int(qr["amount"]) / 10 ** qr["decimals"]
print(f"QR Code for R$ {amount_brl:.2f}")

# Save QR code image
if qr.get("qr_code_base64"):
    img_data = qr["qr_code_base64"].split(",")[1]
    with open("pix_qr.png", "wb") as f:
        f.write(base64.b64decode(img_data))
```

```javascript Node.js
function toCentavos(amount) {
  return (BigInt(Math.round(amount * 100))).toString();
}

const response = await fetch("https://api.bipa.tech/v1/pix/qr-code/static", {
  method: "POST",
  headers: {
    "Authorization": "Bearer bipa_live_sk_...",

  },
  body: JSON.stringify({

    amount: toCentavos(500.00), // R$ 500.00 -> "50000"
    reconciliation_id: "order_12345"
  })
});

const qr = await response.json();
const amountBrl = Number(qr.amount) / 10 ** qr.decimals;
console.log(`QR Code for R$ ${amountBrl.toFixed(2)}`);

// Display in HTML
const img = document.createElement("img");
img.src = qr.qr_code_base64;
document.body.appendChild(img);
```

## QR code types

### Dynamic QR (with amount)
- Fixed payment amount
- Single use
- Expires after payment or timeout
- Best for: invoices, e-commerce checkout

### Static QR (without amount)
- Payer enters amount
- Reusable
- Expires only by timeout
- Best for: donations, tips, general receiving

## Displaying the QR code

You can display the QR code in two ways:

### 1. Using the base64 image

```html
<img src="{qr_code_base64}" alt="Pix QR Code" />
```

### 2. Using the payload with a QR library

```javascript
import QRCode from "qrcode";

// Generate your own QR from the payload
const qrDataUrl = await QRCode.toDataURL(payload);
```

## Handling payments

When someone pays the QR code, a `pix_inflow` event is generated. Poll the [Events API](/api-reference/partner/events/overview) to receive payment notifications:

