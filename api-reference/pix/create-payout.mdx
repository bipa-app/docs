## Request

Initiate a Pix payment from a customer's BRL balance to any Pix key or bank account.
We will only process payouts to counterparties that have the same document as the customer.
If you want to send a payout to a counterparty with a different document, talk to your account manager.


### Body parameters

<ParamField body="customer_id" type="string" required>
  Customer sending the payment
</ParamField>

<ParamField body="amount" type="string" required>
  Amount in centavos (e.g., `"100000"` for R$ 1,000.00)
</ParamField>

<ParamField body="counterparty" type="object" required>
  <Expandable title="counterparty">
    <ParamField body="type" type="string" required>
      Must be `pix_key` or `manual`
    </ParamField>
    <ParamField body="pix_key" type="string">
      The recipient's Pix key (CPF, CNPJ, email, phone, or EVP)
    </ParamField>
    <ParamField body="bank_account" type="object">
      The recipient's bank account (required when type is `manual`).
      <Expandable title="bank_account">
        <ParamField body="ispb_code" type="string" required>
          The recipient's institution ISPB Pix code
        </ParamField>
        <ParamField body="branch_number" type="string" required>
          The recipient's branch number
        </ParamField>
        <ParamField body="account_number" type="string" required>
          The recipient's account number (including the check digit)
        </ParamField>
        <ParamField body="account_type" type="string" required>
          The recipient's account type. Must be one of the following: `checking`, `savings`, `salary` or `payment`.
        </ParamField>
        <ParamField body="name" type="string" required>
          The recipient's name (holder name)
        </ParamField>
        <ParamField body="document" type="string" required>
          The recipient's document (holder document)
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="description" type="string">
  Payment description (max 140 characters). Visible to recipient.
</ParamField>

<ParamField body="idempotency_key" type="string" required>
  Unique key to prevent duplicate payments. Must be unique per API key.
</ParamField>

<Warning>
  The customer must have sufficient BRL balance to complete the payment.
</Warning>

```python Python
import requests
import uuid
from decimal import Decimal

def to_centavos(amount: Decimal) -> str:
    return str(int(amount * 100))

response = requests.post(
    "https://api.bipa.tech/v1/pix/payouts",
    headers={
        "Authorization": "Bearer bipa_live_sk_...",
        "Content-Type": "application/json"
    },
    json={
        "customer_id": "cus_a1b2c3d4e5f6",
        "amount": to_centavos(Decimal("1000.00")),  # R$ 1,000.00
        "counterparty": {
            "type": "pix_key",
            "pix_key": "recipient@email.com"
        },
        "description": "Payment for services",
        "idempotency_key": f"pay_{uuid.uuid4()}"
    }
)

payment = response.json()
amount_brl = int(payment['amount']) / 10 ** payment['decimals']
print(f"Payment status: {payment['status']}, Amount: R$ {amount_brl:.2f}")
```

## Idempotency

Always provide a unique `idempotency_key` to prevent duplicate payments:

```python
# Generate a unique key based on your internal reference
idempotency_key = f"order_{order_id}_payment"

# Or use UUID
idempotency_key = f"pay_{uuid.uuid4()}"
```

If you retry a request with the same idempotency key:
- If the original succeeded, you'll get the original response
- If the original failed, the payment won't be retried automatically

<Note>
  Idempotency keys are valid for 24 hours after the first request.
</Note>
