## Request

Create a Pix key linked to a customer. The key can be used to receive instant payments.

### Body parameters

<ParamField body="customer_id" type="string" required>
  The customer who will own this Pix key
</ParamField>

<ParamField body="type" type="string" required>
  Type of Pix key:
  - `evp` - Random key (recommended)
  - `cpf` - Customer's CPF (individual only)
  - `cnpj` - Customer's CNPJ (business only)
  - `email` - Email address
  - `phone` - Phone number
</ParamField>

<ParamField body="key" type="string">
  The key value. Required for `email` and `phone` types. Auto-generated for `evp`. Must match customer's document for `cpf`/`cnpj`.
</ParamField>

<Note>
  Each customer can have multiple Pix keys, but each key must be unique across the entire Pix network.
</Note>

## Response

<ResponseField name="id" type="string">
  Unique identifier for the Pix key
</ResponseField>

<ResponseField name="customer_id" type="string">
  Customer who owns this key
</ResponseField>

<ResponseField name="type" type="string">
  Key type
</ResponseField>

<ResponseField name="key" type="string">
  The Pix key value
</ResponseField>

<ResponseField name="status" type="string">
  Key status: `pending`, `active`, `inactive`
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp
</ResponseField>

```python Python
import requests

# Create random key
response = requests.post(
    "https://api.bipa.app/v1/pix/keys",
    headers={
        "Authorization": "Bearer sk_live_...",
        "Content-Type": "application/json"
    },
    json={
        "customer_id": "cus_a1b2c3d4e5f6",
        "type": "evp"
    }
)

pix_key = response.json()
print(f"Pix key created: {pix_key['key']}")
```

## Key type restrictions

| Key Type | Customer Type | Notes |
|----------|---------------|-------|
| `evp` | Any | Random UUID, most private option |
| `cpf` | Individual only | Must match customer's CPF |
| `cnpj` | Business only | Must match customer's CNPJ |
| `email` | Any | Must be verified |
| `phone` | Any | Must be verified, E.164 format |
