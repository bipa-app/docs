## Request

Create a unique deposit address for a customer. Each customer can have one address per asset/network combination. Inbound transfers to this address will be automatically credited to the customer's balance.

### Body parameters

<ParamField body="customer_id" type="string" required>
  Customer who will receive inbound transfers
</ParamField>

<ParamField body="asset" type="string" required>
  Asset to receive: `USDT` or `BTC`
</ParamField>

<ParamField body="network" type="string" required>
  Blockchain network:
  - For USDT: `polygon`, `arbitrum`, `optimism`, `tron`, `ethereum`
  - For BTC: `bitcoin`
</ParamField>

<Note>
  If an address already exists for this customer/asset/network combination, the existing address is returned.
</Note>

## Response

<ResponseField name="id" type="string">
  Unique address identifier
</ResponseField>

<ResponseField name="customer_id" type="string">
  Customer who owns this address
</ResponseField>

<ResponseField name="asset" type="string">
  Asset this address accepts
</ResponseField>

<ResponseField name="network" type="string">
  Blockchain network
</ResponseField>

<ResponseField name="address" type="string">
  The deposit address
</ResponseField>

<ResponseField name="status" type="string">
  Address status: `active` or `inactive`
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp
</ResponseField>

```python Python
import requests

# Create USDT deposit address on Polygon
response = requests.post(
    "https://api.bipa.tech/v1/onchain/addresses",
    headers={
        "X-API-Key": "bipa_prod_...",
        "Content-Type": "application/json"
    },
    json={
        "customer_id": "cus_a1b2c3d4e5f6",
        "asset": "USDT",
        "network": "polygon"
    }
)

address = response.json()
print(f"Deposit USDT to: {address['address']}")
print(f"Network: Polygon (MATIC)")
```

## Address reuse

Each customer gets a unique, persistent address per asset/network:

- Calling this endpoint multiple times returns the same address
- Addresses are not shared between customers
- Addresses remain valid indefinitely

<Warning>
  Always verify the network before displaying to users. Sending USDT on the wrong network will result in lost funds.
</Warning>
