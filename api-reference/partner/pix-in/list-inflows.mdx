## Request

Retrieve all PIX inflows (incoming payments) received by users within your partner organization.

## Response

<ResponseField name="pix_in" type="array">
  Array of PIX inflow objects
  <Expandable title="inflow object">
    <ResponseField name="amount_cents" type="integer">
      Payment amount in centavos (BRL smallest unit)
    </ResponseField>
    <ResponseField name="description" type="string">
      Payment description provided by the sender. May be null.
    </ResponseField>
    <ResponseField name="end_to_end_id" type="string">
      Unique PIX transaction identifier (E2E ID)
    </ResponseField>
    <ResponseField name="method" type="string">
      Payment initiation method: `manual`, `dict`, `static_qr_code`, `dynamic_qr_code`, `payer_qr_code`, `initiator`
    </ResponseField>
    <ResponseField name="sender_name" type="string">
      Payer's name
    </ResponseField>
    <ResponseField name="sender_tax_id" type="string">
      Payer's CPF/CNPJ (partially masked for privacy)
    </ResponseField>
    <ResponseField name="sender_account_number" type="string">
      Payer's bank account number
    </ResponseField>
    <ResponseField name="sender_account_type" type="string">
      Payer's account type: `checking`, `savings`, `salary`, `payment`
    </ResponseField>
    <ResponseField name="sender_bank_code" type="string">
      Payer's bank ISPB code
    </ResponseField>
    <ResponseField name="sender_branch_code" type="string">
      Payer's bank branch code
    </ResponseField>
    <ResponseField name="created_at" type="integer">
      Unix timestamp when inflow was created
    </ResponseField>
    <ResponseField name="last_updated_at" type="integer">
      Unix timestamp of last status update
    </ResponseField>
    <ResponseField name="status" type="string">
      Current status: `processing`, `approved`, `denied`, `success`, `credited`, `failed`
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash cURL
curl https://api.bipa.tech/partner/v1/pix-in \
  -H "X-API-Key: your_partner_api_key"
```

```python Python
import requests

response = requests.get(
    "https://api.bipa.tech/partner/v1/pix-in",
    headers={"X-API-Key": "your_partner_api_key"}
)

inflows = response.json()["pix_in"]
for inflow in inflows:
    amount_brl = inflow["amount_cents"] / 100
    print(f"Received R$ {amount_brl:.2f} from {inflow['sender_name']}")
    print(f"  Status: {inflow['status']}")
    print(f"  E2E ID: {inflow['end_to_end_id']}")
```

```javascript Node.js
const response = await fetch("https://api.bipa.tech/partner/v1/pix-in", {
  headers: {
    "X-API-Key": "your_partner_api_key"
  }
});

const { pix_in: inflows } = await response.json();
inflows.forEach(inflow => {
  const amountBrl = inflow.amount_cents / 100;
  console.log(`Received R$ ${amountBrl.toFixed(2)} from ${inflow.sender_name}`);
  console.log(`  Status: ${inflow.status}`);
  console.log(`  E2E ID: ${inflow.end_to_end_id}`);
});
```

```go Go

resp, _ := client.Do(req)
```

## Filtering and pagination

<Note>
  Currently, this endpoint returns all inflows for your partner. For high-volume partners, contact Bipa to discuss pagination and filtering options.
</Note>

## End-to-End ID

The `end_to_end_id` is a unique identifier for each PIX transaction, assigned by the Brazilian Central Bank. It can be used to:

- Track payments across systems
- Reference transactions in support requests
- Reconcile payments with your accounting systems

<Tip>
  Store the `end_to_end_id` for each inflow in your system for easy transaction tracking and reconciliation.
</Tip>
