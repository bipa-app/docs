---
title: "Delete PIX Key"
description: "Delete a PIX key for a partner user."
api: "DELETE https://api.bipa.tech/partner/v1/pix-key/{document}"
---

## Request

Delete the PIX key registered for a user identified by their document (CPF or CNPJ). This operation is asynchronous.

### Path parameters

<ParamField path="document" type="string" required>
  User's CPF (11 digits) or CNPJ (14 digits). Can be formatted or unformatted.
</ParamField>

### Body parameters

<ParamField body="idempotency_key" type="string" required>
  Unique identifier (UUID) to prevent duplicate requests
</ParamField>

## Response

<ResponseField name="status" type="string">
  Deletion status:
  - `accepted` - Deletion request created, processing in progress
  - `already_sent` - Deletion was already requested
  - `succeeded` - Deletion completed
  - `not_found` - PIX key not found for this user
  - `invalid_document` - Document format is invalid
  - `failed` - Deletion failed due to an error
</ResponseField>

```python Python
import requests
import uuid

document = "12345678900"

response = requests.delete(
    f"https://api.bipa.tech/partner/v1/pix-key/{document}",
    headers={
        "X-API-Key": "your_partner_api_key",
        "Content-Type": "application/json"
    },
    json={
        "idempotency_key": str(uuid.uuid4())
    }
)

result = response.json()
print(f"Deletion status: {result['status']}")
```

<Warning>
  Deleting a PIX key is permanent. Once deleted, the user will no longer be able to receive payments using that key. A new key can be created if needed.
</Warning>

## After deleting a PIX key

The deletion is processed asynchronously. Monitor the Events API for a `pix_key` event with `kind: canceled` to confirm the deletion is complete.

<Note>
  It may take a few minutes for the key deletion to propagate through the PIX network. During this time, payments sent to the old key may still be processed.
</Note>
