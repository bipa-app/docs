---

## Request

Retrieve the PIX key registered for a user identified by their document (CPF or CNPJ).

### Path parameters

<ParamField path="document" type="string" required>
  User's CPF (11 digits) or CNPJ (14 digits). Can be formatted or unformatted.
</ParamField>

## Response

<ResponseField name="pix_key" type="string">
  The registered PIX key (EVP/random UUID format)
</ResponseField>

<ResponseField name="created_at" type="integer">
  Unix timestamp when the key was created
</ResponseField>

<RequestExample>
```bash cURL
curl https://api.bipa.tech/partner/v1/pix-key/12345678900 \
  -H "X-API-Key: your_partner_api_key"
```

```python Python
import requests

document = "12345678900"

response = requests.get(
    f"https://api.bipa.tech/partner/v1/pix-key/{document}",
    headers={"X-API-Key": "your_partner_api_key"}
)

if response.status_code == 200:
    data = response.json()
    print(f"PIX Key: {data['pix_key']}")
    print(f"Created: {data['created_at']}")
elif response.status_code == 404:
    print("No PIX key found for this user")
```


## Using the PIX key

Once you have the PIX key, users can receive payments by sharing it with payers. Payers can use the key in their banking app to send instant PIX payments.

The PIX key is an EVP (random key) in UUID format, which provides privacy as it doesn't reveal personal information.

<Tip>
  Display the PIX key to users so they can share it with payers, or use it to generate QR codes for receiving payments.
</Tip>
